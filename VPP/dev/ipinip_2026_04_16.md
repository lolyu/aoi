# SONiC VPP — syncd syslog missing from /var/log/syslog

## Problem

syncd logs do not appear in `/var/log/syslog` on the SONiC VPP host, while swss (orchagent) logs work fine.

## Root Cause

The syncd and swss containers use **different syslog forwarding protocols**:

| Container | Module | Transport | Port |
|-----------|--------|-----------|------|
| swss      | `omfwd` | UDP      | 514  |
| syncd     | `omrelp` | RELP (TCP) | 2514 |

The host rsyslog listens on **UDP 514** (`imudp`) but has **no RELP listener** on port 2514:
- `rsyslog-relp` package is not installed on the host (Debian 13 Trixie)
- `/etc/rsyslog.conf` (Ansible-managed) is missing `module(load="imrelp")` and `input(type="imrelp" ... port="2514")`

The `rsyslog.conf.j2` template in `sonic-buildimage` does include the RELP config, but it was not rendered into the deployed config on this image.

## Runtime Fix (does not survive container restart)

```bash
# Inside the syncd container, replace RELP with UDP forwarding:
docker exec syncd bash -c '
  sed -i "s|module(load=\"omrelp\")|#module(load=\"omrelp\")|" /etc/rsyslog.conf
  sed -i "s|.*action(type=\"omrelp\".*|*.* action(type=\"omfwd\" target=\"127.0.0.1\" port=\"514\" protocol=\"udp\" Template=\"ForwardFormatInContainer\")|" /etc/rsyslog.conf
  supervisorctl restart rsyslogd
'
```

## Permanent Fix (image build)

Install `rsyslog-relp` on the host and ensure `/etc/rsyslog.conf` is generated from the Jinja2 template which includes:

```
module(load="imrelp")
input(type="imrelp" address="127.0.0.1" port="2514")
```

## Verification

```bash
# Confirm syncd logs now appear:
sudo grep syncd /var/log/syslog | tail -5
```
