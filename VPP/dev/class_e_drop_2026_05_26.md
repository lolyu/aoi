# VPP Class E Drop — 2026-05-26

## Background

VPP installs a `240.0.0.0/4` drop route as `FIB_SOURCE_SPECIAL` (highest priority)
in every IPv4 FIB at creation time. Because `FIB_SOURCE_SPECIAL` has the highest
priority, control-plane routes in the class E range cannot override it via the API.
This blocks SONiC-programmed routes in the `240.0.0.0/4` range.

Original approach was a `sonic_fib_init` VPP plugin (PR #232) that removed the route
after startup. Today's work reimplements this as a native VPP startup config option.

## Implementation

**Files changed:**
- `src/vnet/fib/ip4_fib.c`
- `src/vnet/fib/ip4_fib.h`
- `test/test_ip4.py`

### ip4_fib.c changes

1. **Global flag** `int ip4_fib_no_class_e_drop` — zero-initialized, set by config parser

2. **Helper** `static_always_inline ip4_fib_special_is_class_e()`:
   ```c
   static_always_inline int
   ip4_fib_special_is_class_e (const ip4_fib_table_special_prefix_t *s)
   {
       return (s->ift_prefix.fp_addr.ip4.data_u32 == 0xf0000000 &&
               s->ift_prefix.fp_len == 4);
   }
   ```
   No protocol check needed — function is `static`, only receives `ip4_specials[]` entries.

3. **`ip4_fib_hash_load_specials()`** — skip installing `240.0.0.0/4` if flag set:
   ```c
   if (ip4_fib_no_class_e_drop && ip4_fib_special_is_class_e(&ip4_specials[ii]))
       continue;
   ```

4. **`ip4_fib_hash_flush_specials()`** — same skip (can't remove what was never added)

5. **`ip_config()` early config parser** — extended to parse `no-class-e-drop`:
   ```c
   else if (unformat (input, "no-class-e-drop"))
       ip4_fib_no_class_e_drop = 1;
   ```

### ip4_fib.h change

Added extern declaration:
```c
extern int ip4_fib_no_class_e_drop;
```

## Usage

In `/etc/vpp/startup.conf`:
```
ip {
    no-class-e-drop
}
```

Restart VPP. Verify:
```bash
sudo vppctl show ip fib table 0 240.0.0.0/4
# should return nothing if flag is set
```

## Design decisions

| Question | Decision | Reason |
|---|---|---|
| Skip-add vs add-then-remove | **Skip-add** | No wasted FIB ops, no transient drop window |
| Check fp_proto in helper? | **No** | Function is `static`, only called with `ip4_specials[]` (all IPv4) |
| Where to test? | **Merged into `test/test_ip4.py`** | Avoids a separate test file for a small feature |

## Tests (`test/test_ip4.py`)

### `TestIPv4NoClassEDrop`
VPP started with `extra_vpp_config = ["ip", "{", "no-class-e-drop", "}"]`

- `test_class_e_route_absent` — `240.0.0.0/4` must not exist in table 0
- `test_other_specials_present` — `0.0.0.0/0`, `0.0.0.0/32`, `224.0.0.0/4`, `255.255.255.255/32` must still exist
- `test_class_e_absent_in_new_vrf` — create VRF 10, verify `240.0.0.0/4` absent there too

### `TestIPv4ClassEDropDefault`
VPP started with no extra config

- `test_class_e_route_present` — `240.0.0.0/4` must exist by default
