# VPP IPinIP Tunnel — 2026-04-09

## Bug: P2MP Tunnel Lookup Skipped

**Root cause**: `src/vnet/ipip/node.c` fallback order was P2P → 6RD, skipping
`IPIP_MODE_P2MP` entirely. Any P2MP decap fell through to 6RD (no match → drop
with `no tunnel` error counter).

**Fix** (`+7/-2` lines in `node.c`):
```c
ip46_address_reset(&key0.dst);
key0.mode = IPIP_MODE_P2MP;       // ← inserted
t0 = ipip_tunnel_db_find(&key0);
if (!t0) {
    key0.mode = IPIP_MODE_6RD;    // ← fallback preserved
    t0 = ipip_tunnel_db_find(&key0);
}
if (!t0) { goto drop; }
```

Patch file: `0001-ipip-fix-P2MP-tunnel-lookup.patch`

## P2MP Tunnel Bring-Up Sequence (Verified on vlab-vpp-02)

```
vpp# create ipip tunnel src 192.168.0.1 p2mp outer-table-id 0
vpp# set interface state ipip0 up
vpp# set interface ip table ipip0 0
vpp# set interface unnumbered ipip0 use bvi1000
```

**Why each step matters:**

| Command | Why needed |
|---|---|
| `set interface state ipip0 up` | Without this: all RX drops at `ipip0` (interface down) |
| `set interface ip table ipip0 0` | Assigns FIB table membership |
| `set interface unnumbered ipip0 use bvi1000` | Activates ip4 feature arc — without this: `ip4-not-enabled` drops all decapped inner packets |

`bvi1000` (or any interface with an IP in FIB 0) is used as the donor.
`set interface unnumbered` is the trigger that flips `ip4_is_enabled(sw_if_index)` to true.

## Misleading Counters

- **`ipip4-input: packets decapsulated = N`** — incremented unconditionally for
  ALL packets entering the node (line 221 of `node.c`), not only successfully
  decapped ones. Not a reliable success indicator.

- **`ip4-local: unknown ip protocol = N`** — default initial error value set for
  every packet entering `ip4-local` (`error[0] = error[1] = IP4_ERROR_UNKNOWN_PROTOCOL`
  in `ip4_forward.c:1746`). Only actually drops if a registered protocol handler
  is not found (`error != IP4_ERROR_UNKNOWN_PROTOCOL ? DROP : next`). For proto=4
  (IPINIP), the protocol IS registered → dispatched to `ipip4-input` → counter
  fires but packet is NOT dropped. Counter = N is **expected and normal**.

## Drop Sequence Encountered (and Root Causes)

1. **`ipip4-input: no tunnel`** → P2MP lookup bug in `node.c` (fixed)
2. **`ipip0: drops, state=down`** → `set interface state ipip0 up` missing
3. **`ipip0: drops, state=up, MTU=9000/0/0/0`** → `set interface ip table` missing
4. **`ip4-not-enabled`** → `set interface unnumbered` missing

## ECMP with IPinIP Inner 5-Tuple

- **After decap** (tunnel terminates here): ✅ Works automatically. VPP hashes on
  inner packet 5-tuple via `ip4_compute_flow_hash()` after `ipip4-input` strips
  the outer header.
- **Transit** (forward without decap): ❌ Not supported in SONiC-VPP. VPP hashes
  on outer header only. `saivpp` does not implement `SAI_OBJECT_TYPE_FINE_GRAINED_HASH_FIELD`
  so PBH-based inner hash is not available.
