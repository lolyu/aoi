# VPP MP2P

## Goal

Add a new IPIP tunnel mode `MP2P` (multi-point-to-point, decap-only) to VPP and
wire SONiC's `SAI_TUNNEL_TERM_TABLE_ENTRY_TYPE_P2MP` to it. Old `MP` mode requires
TEIB/NBMA next-hops; `MP2P` doesn't because it never encaps.

## VPP changes (gerrit 45685)

Files:
- `src/vnet/tunnel/tunnel.h` — add `TUNNEL_MODE_MP2P` to `foreach_tunnel_mode`.
- `src/vnet/tunnel/tunnel_types.api` — add `TUNNEL_API_MODE_MP2P [backwards_compatible]`
  so the new enum value doesn't bump CRCs of any message embedding `tunnel_mode`
  (ipip / gre / ipsec_itf / ipsec_sa\*).
- `src/vnet/tunnel/tunnel.c` — `unformat_tunnel_mode` recognizes `mp2p`
  (placed before `mp` so the prefix doesn't shadow it).
- `src/vnet/ipip/ipip.h` `ipip.c` — add `IPIP_MODE_MP2P`. API rejects non-zero `dst`
  (same rule as MP). Binds the simple P2P HW interface class (encap rewrite unused),
  not the `mipip` NBMA class — so no TEIB entries needed.
- `src/vnet/ipip/ipip.c` (decap lookup) — extend lookup to also match MP2P keys
  (P2P → MP2P → 6RD).
- `src/vnet/ipip/ipip_cli.c` — `show ipip tunnel` formats MP2P; `create ipip tunnel`
  short_help adds `[mp2p]`.
- `test/test_ipip.py` — MP2P decap from multiple peers; reject non-zero `dst`.

CLI:
```
vpp# create ipip tunnel src 10.1.0.32 mp2p
vpp# show ipip tunnel
```

API: `ipip_add_tunnel` with `tunnel.mode = TUNNEL_API_MODE_MP2P (=2)` and zero `dst`.

## Gerrit gate fixes

- **checkstyle**: clang-format-15 reformatted the `foreach_tunnel_mode` macro,
  the `TUNNEL_MODE_MP||MP2P` if-condition in `ipip.c`, the `vnet_register_interface`
  call, and the MP2P `format(...)` line in `ipip_cli.c`. Apply only those targeted
  reflows (don't run clang-format over whole files).
- **CRC checker**: any non-`[backwards_compatible]` enum addition bumps CRCs of
  every message that embeds `tunnel_mode` → 15+ APIs flagged. Fix = mark
  `TUNNEL_API_MODE_MP2P [backwards_compatible]`.

`[backwards_compatible]` semantics: VL API generator emits the field but excludes
it from the CRC. Only valid for additive changes (new trailing field, new enum
value). Old clients keep working because they never set/observe the new value.

## ret=-8 (`VNET_API_ERROR_INVALID_VALUE_2`)

Returned by `tunnel_mode_decode()` when the api mode value isn't in
`foreach_tunnel_mode`. Means: running VPP doesn't know `MP2P=2` (old deb), but
syncd was rebuilt with the new enum → mismatch.

Verify the binary contains MP2P:
```
strings /usr/bin/vpp /usr/lib/x86_64-linux-gnu/vpp_plugins/*.so 2>/dev/null \
  | grep -c multi-point-to-point     # >= 1
vppctl show version | grep b1sonic   # should show the bumped suffix
```

## Force a local VPP rebuild in sonic-buildimage

Background: `platform/vpp/vppbld/Makefile` does
`curl -L -f $(VPP_DEB_URL)/vpp_$(VPP_VERSION_SONIC)_amd64.deb || build_locally`.
With the cached version, curl succeeds → **patch is skipped**. Need curl to 404.

Steps:

1. **Bump `VPP_VERSION_SONIC`** in `platform/vpp/rules/vpp.mk` so the buildkite
   URL 404s, e.g. `2510-0.2+b1sonic1` → `2510-0.2+b1sonic2`. ("Bump" = increment
   the version string; the deb filename embeds it, so a new value invalidates the
   remote cache.)
2. Drop the patch into `platform/vpp/vppbld/patches/` and append its name to
   `patches/series`. `repo_clone` runs `quilt push -a` against the freshly cloned
   VPP tree.
3. Remove stale debs + downstream docker/img so make re-derives them:
   ```bash
   cd vpp_build2/sonic-buildimage
   rm -f target/debs/bookworm/{libvppinfra,vpp,python3-vpp-api}*.deb \
         target/debs/bookworm/{libsairedis,libsaivpp,libsaivs,syncd-vpp,syncd-vs,python3-pysairedis}*.deb
   rm -f target/docker-syncd-vpp.gz target/docker-gbsyncd-vpp.gz \
         target/docker-sonic-vpp.gz target/sonic-vpp.bin target/sonic-vpp.img.gz
   ```
4. `make <sonic-vpp target>`. Verify:
   ```bash
   ls target/debs/bookworm | grep b1sonic2
   dpkg-deb --fsys-tarfile target/debs/bookworm/vpp_2510-0.2+b1sonic2_amd64.deb \
     | tar -xO ./usr/bin/vpp | strings | grep -c multi-point-to-point   # >=1
   ```

Why downstream debs too: sairedis/syncd `#include` the VPP api headers, so they
must be recompiled to pick up `TUNNEL_API_MODE_MP2P`. `MAIN_TARGET = libvppinfra…deb`
in `vppbld/Makefile` gates the whole VPP package set.

Alternatives if version bump isn't possible: comment out the `curl` line in
`vppbld/Makefile`, or build debs manually in `vpp_build2/vpp/` (`make pkg-deb`)
and copy them into `target/debs/bookworm/` with matching filenames.
