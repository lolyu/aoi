# VPP Tunnel Encap CLI Reference

## IPIP Tunnel

```
vpp# create ipip tunnel src 10.1.0.32 dst 10.1.0.33 outer-table-id 0
vpp# set interface state ipip12 up
vpp# set interface ip table ipip12 0
vpp# set interface unnumbered ipip12 use loop0
vpp# ip route add 193.11.232.128/25 via ipip12
```

### Encap/Decap Flags

| Flag | Hex | Meaning |
|---|---|---|
| `ENCAP_COPY_DF` | 0x01 | Copy DF bit from inner |
| `ENCAP_SET_DF` | 0x02 | Set DF bit on outer header |
| `ENCAP_COPY_DSCP` | 0x04 | Copy DSCP from inner to outer |
| `ENCAP_COPY_ECN` | 0x08 | Copy ECN from inner to outer |
| `DECAP_COPY_ECN` | 0x10 | Copy ECN from outer to inner |
| `ENCAP_COPY_HOP_LIMIT` | 0x40 | Copy TTL/hop-limit from inner |

## VxLAN Tunnel

```shell
# Basic unicast VxLAN
create vxlan tunnel src 1.0.0.1 dst 1.0.0.2 vni 3000

# With custom ports
create vxlan tunnel src 1.0.0.1 dst 1.0.0.2 vni 3000 src_port 59000 dst_port 59001

# With encap VRF
create vxlan tunnel src 10.0.3.1 dst 10.0.3.3 vni 13 encap-vrf-id 7

# Multicast VxLAN
create vxlan tunnel src 10.0.3.1 group 239.1.1.1 GigabitEthernet0/8/0 vni 13

# L3 mode (routed, not bridged)
create vxlan tunnel src 1.0.0.1 dst 1.0.0.2 vni 3000 l3

# L2 bridging setup (typical SONiC VNet use)
set interface l2 bridge vxlan_tunnel0 3000 1
bvi create mac 00:00:00:00:00:01
set interface l2 bridge bvi0 3000 bvi

# Add neighbor + route over tunnel
ip neighbor vxlan_tunnel0 1.0.0.2 00:00:00:00:00:01 no-fib-entry
ip route add 100.1.1.0/24 via 1.0.0.2 vxlan_tunnel0

# Delete / Inspect
create vxlan tunnel src 1.0.0.1 dst 1.0.0.2 vni 3000 del
show vxlan tunnel
```

## SONiC → VPP Mapping

### VxLAN (via TunnelManager::tunnel_encap_nexthop_action)

SONiC config:
```json
// CONFIG_DB
"VXLAN_TUNNEL": { "test": { "src_ip": "1.0.0.1" } }
"VNET": { "Vnet1": { "vni": "3000", "vxlan_tunnel": "test" } }
// APPL_DB
"VNET_ROUTE_TUNNEL_TABLE:Vnet1:100.1.1.0/24": { "endpoint": "1.0.0.2" }
```

VPP calls: `create_vpp_vxlan_encap()` + `create_vpp_vxlan_decap()`

### IPIP (via TunnelManagerIpIp::ipip_encap_nexthop_action)

Flow:
1. `vpp_ipip_tunnel_add()` — create the IPIP tunnel
2. `interface_set_state` UP
3. `set_interface_vrf` — assign overlay VRF
4. `sw_interface_set_unnumbered` — borrow IP from existing interface

Flags mapped from SAI attrs via `TunnelManagerIpIp::map_sai_to_vpp_flags()`:
- `SAI_TUNNEL_ATTR_DECAP_ECN_MODE` → `DECAP_COPY_ECN`
- `SAI_TUNNEL_ATTR_DECAP_TTL_MODE` (uniform) → `ENCAP_COPY_HOP_LIMIT`
- `SAI_TUNNEL_ATTR_DECAP_DSCP_MODE` (uniform) → `ENCAP_COPY_DSCP`
- `SAI_TUNNEL_ATTR_ENCAP_ECN_MODE` (user_defined) → `ENCAP_COPY_ECN`

### Key code paths

- `TunnelManager::get_tunnel_if()` — looks up VxLAN first, falls through to IPIP
- `TunnelManager::tunnel_encap_nexthop_action()` — dispatches by `SAI_TUNNEL_ATTR_TYPE`
  - `SAI_TUNNEL_TYPE_VXLAN` → VxLAN path
  - `SAI_TUNNEL_TYPE_IPINIP` → `m_tunnel_mgr_ipip.ipip_encap_nexthop_action()`
- `SwitchVpp::create()` for `SAI_OBJECT_TYPE_TUNNEL_TERM_TABLE_ENTRY` → `m_tunnel_mgr_ipip.create_ipip_tunnel_term()`
